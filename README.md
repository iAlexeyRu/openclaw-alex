# openclaw-alex

Живой workspace Алисы для Алекса.

Что здесь хранится:
- identity и persona-файлы (`IDENTITY.md`, `SOUL.md`)
- профиль Алекса (`USER.md`)
- правила и устройство workspace (`AGENTS.md`)
- локальные заметки по инструментам (`TOOLS.md`)
- heartbeat-настройки (`HEARTBEAT.md`)
- локальные skill'ы (`skills/`, см. `skills/INDEX.md`)
- память и рабочие записи (`memory/`, `MEMORY.md`)
- другие полезные файлы, которые Алиса создаёт по ходу работы

## Принцип

Это не только стартовый шаблон. Это рабочий репозиторий, в котором Алиса хранит свой актуальный workspace и ведёт его вместе с Алексом.

## Что храним в git

- основные persona и workspace-файлы
- `MEMORY.md` и содержимое `memory/`, если там нет чувствительных данных
- избранное служебное состояние, если оно полезно для переносимости workspace
- сейчас из `.openclaw/` храним только `.openclaw/workspace-state.json`

## Что обычно не коммитится

- прочее содержимое `.openclaw/`, если это runtime-state, кэш или локальная служебка
- приватные секреты и токены, если только Алекс не попросил сохранить что-то конкретное осознанно
- временный мусор и кэш

## Осторожно с секретами

По умолчанию секреты в репо не храним. Если что-то чувствительное всё же сохранено по прямой просьбе Алекса, считаем это исключением, а не нормой.
