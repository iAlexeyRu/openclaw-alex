Алекс пишет в Telegram, Алиса внутри OpenClaw работает с навыками, git и веб-инструментами.

Когда ставить architecture-diagram

  • • Нужно красиво показать сервисы, базы, очереди и внешние интеграции
  • • Хочешь не скетч, а почти слайд для доки или презентации
  • • Нужен один HTML без тяжёлых зависимостей

Когда ставить systematic-debugging

  • • Агент начинает гадать и предлагать фиксы на удачу
  • • Есть flaky test, regression или интеграционный баг
  • • Нужно заставить расследовать причину, а не метаться

Когда ставить codebase-inspection

  • • Хочешь быстро понять размер репы и стек
  • • Нужно сравнить два репозитория по объёму
  • • Надо посчитать LOC без ручной ерунды
